# ProductionWorkshop

Симулятор работы производственного цеха. Программа моделирует прохождение изделий через технологические этапы (типы обработки) на различных станках с учетом их текущей загрузки, очередей и времени обработки.

---

## Описание модели

Производственная система состоит из $M$ типов состояний изделий (где состояния $0 \dots M-2$ соответствуют технологическим этапам обработки, а состояние $M-1$ означает готовность изделия) и $N$ станков.

1. **Технологическая матрица $T_{i,j}$** (размера $(M-1) \times N$) определяет время обработки изделия типа $i$ на станке $j$.
2. **Маршрутизация**: При переходе на следующий этап изделие направляется на станок с наименьшей прогнозируемой загрузкой (суммарное время выполнения всех запланированных на нем задач). При равенстве приоритет отдается станку с меньшим индексом.
3. **Очереди**: На каждом станке есть входной буфер (`incoming_box`). Если станок занят, прибывшее изделие ожидает в очереди.

---

## Жизненный цикл изделий и лог событий

В процессе работы симулятор выводит в консоль лог переходов в хронологическом порядке. Формат вывода событий:

* `start <time> <product_id> <type_id> <machine_id>` — начало обработки изделия на станке.
* `finish <time> <product_id> <type_id> <machine_id>` — завершение обработки изделия на станке.
* `wait <time> <product_id> <type_id> <machine_id> <queue_size>` — перемещение изделия в очередь станка (где `queue_size` — количество изделий перед ним).
* `ready <time> <product_id> <machine_id>` — изделие успешно прошло все этапы обработки (достигло типа $M-1$).
* `stop <time>` — завершение работы всего цеха (события обработаны).

### Порядок вывода при совпадающем времени

Если несколько событий происходят в один и тот же момент времени `time`, они упорядочиваются согласно следующим правилам (в порядке убывания приоритета):
1. **Тип события**: `finish` $\rightarrow$ `start` $\rightarrow$ `wait` $\rightarrow$ `ready`.
2. **Индекс станка**: При совпадении типа события сортировка производится по возрастанию номера станка (`machine_id`).

## Заметки по расхождениям в тестовых примерах

В некоторых спецификациях (например, демонстрационных PDF) могут встречаться нестыковки в расчетах времени и порядке вывода событий при их совпадении:

1. **Несоответствие времени завершения в примерах**:
   * Для изделия $k=3$ на Станке 1 время завершения должно рассчитываться как $0 + T_{1,1} = 6$, однако в некоторых примерах указано значение $4$.
   * Для изделия $k=1$ на Станке 0 время завершения должно составлять $3 + T_{1,0} = 7$, в то время как в примере приводится значение $5$.
   * Данный симулятор производит строгий расчет времени на основе переданной матрицы $T_{i,j}$.

2. **Порядок вывода при совпадающем времени**:
   В ряде примеров порядок вывода событий при одинаковом значении `time` может выглядеть хаотично. В данной реализации логика сортировки приведена к строгому стандарту: `finish` $\rightarrow$ `start` $\rightarrow$ `wait` $\rightarrow$ `ready`, что гарантирует детерминированность вывода.

## Формат входного файла

Входной текстовый файл должен иметь следующую структуру:

1. Первая строка: два целых числа $M$ ($1 \le M \le 100$) и $N$ ($1 \le N \le 100$).
2. Следующие $M-1$ строк: по $N$ целых чисел в каждой — элементы матрицы $T_{i,j}$ ($0 \le T_{i,j} \le 10000$).
3. Следующие $N$ строк: описание начальных очередей для каждого станка (от $0$ до $N-1$). Каждая строка содержит:
   * Количество изделий в очереди на данном станке.
   * Перечисление начальных типов изделий (целые числа от $0$ до $M-2$).

### Пример входных данных:
```txt
3 2
6 4
5 3
2 0 1
1 0
```

## Сборка проекта

Для сборки проекта необходим компилятор с поддержкой стандарта C++20 и установленный CMake (версии 3.16 или выше).

```bash
# 1. Создание директории сборки и генерация конфигурации
cmake -S . -B build -DCMAKE_BUILD_TYPE=Release

# 2. Компиляция проекта
cmake --build build
```

После завершения сборки исполняемый файл `production_workshop` будет создан в каталоге `build`.

---

## Запуск

Программа принимает в качестве аргумента путь к входному файлу конфигурации:

```bash
./build/production_workshop <path_to_input_file>
```

Результат симуляции (события переходов и финальное время работы) выводится непосредственно в стандартный поток вывода (`std::cout`).
